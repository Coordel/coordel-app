Add Task
This is the main button that the user sees at the top left of the page. It's should be clear that it's an add button
 
Edit
This is the button that is clicked to Edit the current object. It can appear in the header of the Coord or Task.
 
Actions
This button shows a menu with all the Actions a user can take in the current context. So if they are working on Task details they will see one menu, and if they are working on a Coord, they will see another menu
 
Notifications
This button has two states. Notifications arrive when someone else makes a change that affects the user. This one indicates that there are no new Notifications.
Notifications Active
This is the second state. It will show the number of notifications on the blue background
 
Sort
When the user clicks this icon, they see a menu with options for sorting and grouping. They can sort by Deadline, Date Created, and Date updated. They can group by Coord, by Timeline. They can opt to sort descending/ascending and they can choose whether or not to show the checklist for Tasks. Needs to have a disabled state when not available.
 
Go Turbo
When the user clicks Turbo, they go into the Getting Things Done wizard
 
Cancel Turbo
This stops the GTD Turbo Wizard
 
Done
This is the button that the user clicks to mark a Task or Coord done. It needs to have disabled state when it's not possible to mark done.
 
Back
This will have the place from where the user came after the icon. So if they clicked to got to Task Details in the Current list then there would be the back icon and the word Current after it in the button
 
Hide
Hides the right column of the interface and toggles to show
 
Show
Shows the right column of the interface and toggles to hide
 
Accept
Clicking this button Accepts an object based on context: Coord or Task
 
Decline
Clicking this button Declines an object based on context: Coord or Task
 
Send
Clicking this button makes a Pending Coord Active for everyone
